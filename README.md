Formal verification project for a FIFO hardware model using **NuSMV** and temporal logic assertions.  
This project was developed as part of the **Functional Verification of HDL Models** course at the **University of Tehran** under the supervision of **Dr. Siamak Mohammadi**.

The project focuses on:

- Writing and validating **LTL** and **CTL** properties
- Performing **model checking** on a FIFO design
- Analyzing counterexamples generated by NuSMV
- Verifying correctness and behavioral consistency of FIFO operations

---

# Project Overview

This repository contains a formal verification workflow for a FIFO hardware module implemented in **SMV format** and verified using the **NuSMV model checker**.

The verification process includes:

- FIFO behavioral modeling
- Temporal property specification
- LTL and CTL assertion development
- Formal model checking
- Counterexample analysis
- Verification report and documentation

The project demonstrates how formal methods can be used to mathematically verify hardware correctness instead of relying solely on simulation-based testing.

---

# Verification Topics

## LTL Properties

The following Linear Temporal Logic (LTL) properties were verified:

- Mutual exclusion of `full` and `empty`
- FIFO eventually becomes full under continuous writes
- Data written into FIFO eventually appears at output
- Verification using frozen variables
- Empty signal consistency during write/read operations
- Sequential data ordering validation

## CTL Properties

Several Computation Tree Logic (CTL) properties were also analyzed, including:

- Reachability checks
- Full-to-empty state transitions
- Path-based behavioral validation
- Counterexample generation for invalid specifications

# Running the Verification

## 1. Install NuSMV

Download NuSMV from the official website:

https://nusmv.fbk.eu/

## 2. Run Model Checking

Execute the following command in Linux:

```bash
NuSMV fifo.smv
```

Or preprocess and execute:

```bash
NuSMV -pre cpp fifo.smv
```

# Sample Verification Results

The verification process produces:

- PASS results for satisfied properties

- Counterexample traces for debugging invalid assertions

The generated traces help analyze FIFO behavior under different temporal conditions.
